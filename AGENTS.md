# Project: Overseas Remittance Mock (Spec v0.2)
## Goal
- 購買部の海外送金業務向けに**最小モック**を実装して先方に動作を見せる。
- 動作の認識合わせをすることが目的であるため、各種疎通などの精度は求めない。アプリケーションとして見せれるモックがあることが最優先。特に「確認と修正」部分
- **Azure Document Intelligenceとは疎通しない**。代わりに**擬似抽出(JSON)**で処理。
- 出力は**経理指定の固定Excelテンプレート**（exceljsで生成）。手動DLでOK。（まずは仮で良く、精度は求めない）

## Scope (Must)
- フロント：単一ページ
  - PDFファイル(単票/複数)のドラッグ&ドロップ
  - 抽出結果フォーム（支払金額/通貨/支払先/品目…）と**簡易エラー表示**
  - 「Excel出力」ボタン（擬似データ→Excel生成→ブラウザDL）
- バック：最小API
  - `POST /api/import` : PDF受取→(ハッシュ/重複検知はスキップ可)→**擬似抽出**返却
  - `POST /api/export` : JSON→Excelテンプレ埋め込み（exceljs）
- ストレージ：**ローカル `./data`**（開発用）。ログはコンソールのみ。

## Non-Goals (Not now)
- 認証・権限/閉域/本番インフラ設計
- 楽々DP API連携
- 本番OCR連携（Azure DI/Gemini/GCP/AWS）

## **実装イメージ**

担当者が直感的に利用できる、セキュアなWebアプリケーションを想定しています。

- **Step 1: アップロード**
    - Web画面に請求書PDFをドラッグ＆ドロップ（単票／バッチ）。
    - アップロード時にファイル指紋（ハッシュ）生成・重複検知・メタ情報（推定通貨・取引先候補）付与。
    - オブジェクトストレージへ保存（非公開・暗号化）
- **Step 2: AIによる自動抽出・転記**
    - AI-OCRで解析し、抽出結果（値・信頼度・位置情報）を取得
- **Step 3: 確認と修正（必要なら対応。運用は要相談）**
    - 1**次レビュー**
        - 画面は**左：PDFビューア（抽出箇所ハイライト）／右：Excelテンプレのプレビュー**＋下部に**検証エラー/警告パネル**
        - 要確認リストの低信頼・検証NG項目を中心に修正し、1次確定
    - **2次レビュー**
        - 特定の条件（高額・初取引など特に注力する対応）は実施
        - 差し戻しなどできるイメージ
- **Step 5: 出力と申請**
    - 「Excel出力」ボタンをクリックで、固定Excelテンプレートへデータ埋め込み → **報告書Excel（確定版）を生成。**
    - ファイルをダウンロードし社内システム（楽々DP）へ添付
    - （※楽々DPのAPI連携が可能であれば、この添付作業も自動化を目指します）

## Tech Baseline
- Next.js + TypeScript（App Router） / Node 18+
- パッケージ：`exceljs`（Excel生成）
- UIはシンプル（PDFプレビューは**任意**。フォーム＋エラーパネルは**必須**）
